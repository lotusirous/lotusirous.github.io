Build a go web application pipeline

Build and deploy web application is necessary for any organization who uses golang as a main language to build their bushiness. This article summarized ways to build and deploy a golang application from a simple to a complex pipeline. The decision depends on architecture designer’s style. However, I recommend to follow 2 principles when designing a system: keep it simple and stupid (KISS) because you aren’t gonna need it (YAGNI).

Our application

Before we begin, I created a simple dna strand conversion web api service in this repository. The api follows a query ?q= with a DNA string ATT.

GET localhost:8080/?q=ATT HTTP/1.1

HTTP/1.1 200 OK
Content-Length: 17
Content-Type: application/json

{
  "result": "TAA"
}

A manually way

In case, you work alone or you are very lazy to build a complex system. The repository included a Makefile build and deploy to a single machine as follows:

.PHONY: build test manual

build:
    go build -o app

run:
    ./app

test:
    go test -race -v ./...

manual: test build run

clean:
    rm -rf app

Now, you can build your application by command make manual to run your application and make clean to clean up previous file. Easy, huh ?

You can see a disadvantages of this way is updating application manually by executing make command since it does not automatically update your application.

The story can not be ended here, we have to configuration a process management and log rotate for our program. In this case, I use supervisord to run our go program. The supervisord’s configuration can be found in deployment directory.
